# Part-1 Exam Preview

ကျွန်တော်တို့တွေက ကိုယ်ဖြေမယ့် စာမေးပွဲရဲ့ အကြောင်းအရာကိုတော့သိထားသင့်ပါတယ်။ အဲ့တာမှလဲ ကိုယ်က ဘယ်လိုလေ့လာရမယ် ဘာတွေကိုလေ့ကျင့်ရမလဲ ကိုယ်က ဘယ်အပိုင်းတွေမှာ အားနည်းလဲ စတာတွေကို သေချာ သိနိုင်မှာာ ဖြစ်ပါတယ်။ ဒါဆို စလေ့လာရအောင်။

### 1.1.1 Exam Format

AWS SAA-C02 Exam ဖြေပြီဆိုရင် ကျွန်တော်တို့ဟာ စာမေးပွဲခန်းထဲမှာ မေးခွန်း 65 ပုဒ်တိတိကို ရင်ဆိုင်ဖြေဆိုရမှာဖြစ်ပါတယ်။ ဒီ 65 ပုဒ်ရဲ့ စုစုပေါင်းအမှတ်ကတော့ 1000 ဖြစ်တယ်။ အောင်မှတ်ကတော့ 720 ဖြစ်ပါတယ်။ ကျွန်တော်တို့တွေက အနည်းဆုံး အမှတ် 720 ရမှာ အောင်နိုင်ဖြစ်တယ်လို့ မှတ်ထားရပါတယ်။

ဒီနေရာမှာ ထူးခြားချက် တစ်ခု ရှိပါသေးတယ်။ AWS က ကျွန်တော်တို့ဖြေဆိုမယ့် 65 ပုဒ်လုံးကို အမှတ်ပေးမှာမဟုတ်ပါဘူး။ 65 ပုဒ်ထဲက 15 ပုဒ်ကိုတော့ အမှတ်ပေးတဲ့နေရာမှာ ရေတွက်မှာ မဟုတ်တာပါပဲ။ ဒီ 15 ပုဒ်ကလဲ ဘယ်ဟာတွေလဲ ဆိုတာကို လဲ ကျွန်တော်တို့သိနိုင်မှာ မဟုတ်ပါဘူး။

စာမေးပွဲဖြေဆိုချိန်အနေနဲ့တော့ 130မိနစ်ပဲ ဖြစ်ပါတယ်။ ဒါပေမယ့် ကျွန်တော်တို့လို မျက်နှာဖြူစကား အရမ်းမကျွမ်းတဲ့ လူတွေအတွက်ကတော့ အပိုဆောင်း အနေနဲ့ ၃၀ မိနစ် ကို ထပ်ပေးထားပါသေးတယ်။ အဲ့တော့ ကျွန်တော်တို့စာမေးပွဲဖြေမယ်ဆိုရင် ၁၆၀မိနစ် ဖြေဆိုချိန်ရမှာဖြစ်ပါတယ်။

**160 / 65 = 2.4** ဆိုတော့ မေးခွန်းတစ်ခုခြင်းစီကို ဖြေဆိုချိန်ဟာ ၂မိနစ်ခွဲလောက်ပဲ အချိန်ရတာလို့ အဓိပ္ပာယ်ရပါတယ်။

နောက်တစ်ခုကတော့ မေးခွန်းအမျိုးအစားတွေဖြစ်ပါတယ်။ lab လုပ်ရတဲ့ မေးခွန်းအမျိုးအစားတွေတော့မပါပါဘူး။ Multiple Choice လို့ခေါ်တဲ့ အဖြေတွေအများကြီးက အဖြေမှန်တစ်ခုကိုရွေးပေးရတယ် မေးခွန်းမျိုးရယ် Multiple Responce လို့ခေါ်တဲ့ တစ်ခုထက်ပိုတဲ့ အဖြေကိုရွေးခိုင်းတဲ့ မေးခွန်းမျိုးတွေပဲ ပါဝင်မှာဖြစ်ပါတယ်။

စာမေးပွဲကြေးအနေနဲ့ ဒေါ်လာ ၁၅၀ ကျသင့်မှာဖြစ်ပါတယ်။ တစ်ကယ်လို့ကိုယ်က စာမေးပွဲအောင်သွားခဲ့ရင်တော့ နောက်ဖြေမယ့် AWS စာမေးပွဲအတွက် 50% သက်သာစေမယ့် Voucher ကိုရရှိမှာဖြစ်ပါတယ်။

ဒါကတော့ AWS SAA-C03 Exam ရဲ့ စာမေးပွဲအနှစ်ချုပ်ပဲဖြစ်ပါတယ်။ ဆက်ပြီးတော့ စာမေးပွဲမှာ ပါဝင်မယ့် Domain တွေကိုကြည့်ကြည့်ရအောင်။

## 1.1.2 Domain Weight

စာမေးပွဲမှာကတော့ AWS ရဲ့ မြောက်မြားစွာသော Service တွေကို ဖြေဆိုရမှာဖြစ်ပါတယ်။ ဒါပေမယ့် ကျွန်တော်တို့က အနှစ်ချုပ်ကြည့်လိုက်မယ်ဆိုရင်တော့ Domain 4 ခု ကိုပဲ ရမှာ ဖြစ်ပါတယ်။

    Domain 1: Design Secure Architectures           → 30%
    Domain 2: Design Resilient Architectures        → 26%
    Domain 3: Design High-Performing Architectures  → 24%
    Domain 4: Design Cost-Optimized Architectures   → 20%

ဒီဇယားလေးကို ကြည့်လိုက်မယ်ဆိုရင်တော့ Security ပိုင်းကို အများဆုံးမေးတာကိုတွေ့ရမှာဖြစ်ပြီးတော့ Cost Optimization အပိုင်းကိုတော့ အနည်းဆုံးမေးတာကိုတွေ့ရမှာပါ။ ဒါပေမယ့် အများဆုံးနဲ့ အနည်းဆုံးကြားထဲမှာ 10% ကွာခြားချက်ပဲရှိတာကြောင့်မလို့ Domain အကုန်လုံးကို ထုံးလိုကြေ ရေလိုနောက် ကြေညှက်ထားဖို့လိုအပ်ပါတယ်။

## 1.1.1 Scoring

AWS က Domain တိုင်းကိုအောင်မှ ရမယ်လို့ သတ်မှတ်ထားတာမဟုတ်ပါဘူး။ Domain 4ခုပေါင်းကို 720မှတ်ကျော်ရင် အောင်ပေးမှာဖြစ်ပါတယ်။
